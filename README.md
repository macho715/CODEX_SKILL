experiments, and project-specific automation bundles.

- [mstack-codex-package-1.1.0](mstack-codex-package-1.1.0):

## Workspace Layout

- `docs/`: managed project docs and workspace-level notes
- `plugins/`: shared plugin-related material
- `.agents/`: agent-local support files
- `_holding-root/`: non-destructive archive area for moved temp files, zip
  exports, loose docs, and legacy package exports

## Notes

- This root `README.md` is the human entrypoint for the workspace.
- Project-specific install, validation, and runtime instructions stay inside
  each project folder.
- Managed documentation generated by `project-doc-orchestrator` stays under
  `docs/project-docs/`.
