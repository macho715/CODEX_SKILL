# Review Checklist

## Core structure
- [ ] 문서 최상위 제목이 분명한가
- [ ] Heading 계층이 논리적인가
- [ ] 제목과 바로 아래 문단의 연결이 자연스러운가

## Paragraph
- [ ] 한 문단에 한 메시지 원칙이 지켜지는가
- [ ] 빈 줄 반복으로 간격을 만들지 않았는가
- [ ] 본문 기본 정렬이 좌측 정렬인가
- [ ] 번역투/명사형 과밀/중복 표현이 줄었는가

## Font
- [ ] Latin/English = Aptos 정책이 기록되었는가
- [ ] East Asian/Korean = 맑은 고딕 정책이 기록되었는가
- [ ] plain text 환경이면 run-level font를 확정하지 않았는가

## Table
- [ ] 표 제목이 표 위에 있는가
- [ ] 헤더 행이 있는가
- [ ] 텍스트/숫자/날짜 정렬이 적절한가
- [ ] 셀 병합과 중첩 표를 최소화했는가
- [ ] repeated header / row split 항목이 필요하면 AMBER로 분리했는가

## Accessibility
- [ ] 표 구조가 단순한가
- [ ] 링크/캡션/설명이 의미 중심인가
- [ ] 과도한 색상/밑줄/이탤릭이 제거되었는가

## Preservation
- [ ] 날짜/숫자/금액/통화/ID를 변경하지 않았는가
- [ ] 계약 조항 번호, annex 번호, 표 번호를 보존했는가
