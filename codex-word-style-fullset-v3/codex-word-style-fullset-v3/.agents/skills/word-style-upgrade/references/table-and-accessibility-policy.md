# Table and Accessibility Policy

## Table structure
- 표 제목은 표 위에 둔다
- 표 헤더는 필수
- 1셀 1정보를 기본으로 한다
- 셀 병합과 중첩 표는 최소화한다

## Alignment rules
- 텍스트: 좌측 정렬
- 숫자/금액/%: 우측 정렬
- 날짜: 중앙 또는 우측 정렬
- 상태/No/Rev: 중앙 정렬

## Size and borders
- 표 너비는 페이지 폭 95~100% 이내
- 너무 넓으면 글자 축소보다 열 재설계를 먼저 검토
- 바깥선 0.75~1.00pt
- 안쪽선 0.25~0.50pt
- 헤더행은 Bold + 연회색 음영

## Accessibility
- heading order를 논리적으로 유지
- simple table 유지
- 의미 있는 링크 텍스트와 캡션 사용
- Accessibility Checker로 최종 점검

## AMBER layout checks
plain text 환경이면 아래를 확인 불가로 둘 수 있다.
- Repeat Header Rows
- Allow row to break across pages
- exact cell padding
- actual page overflow
