# Generic Verification Checklist

Use this checklist when the draft is not claiming a planner-specialist-verifier workflow.

## Required checks

### 1. Objective clarity
PASS if the draft states the task, decision, or workflow objective clearly.
AMBER if the objective is implied but not explicit.
FAIL if the reader cannot tell what the draft is trying to achieve.

### 2. Acceptance criteria or success conditions
PASS if success conditions, constraints, or completion boundaries are visible.
AMBER if they exist but are vague.
FAIL if the draft claims completion without saying what good looks like.

### 3. Evidence and assumptions separation
PASS if evidence-backed statements and assumptions are clearly separated.
AMBER if assumptions are mixed into findings but still inferable.
FAIL if unsupported claims are presented as facts.

### 4. Gaps and blockers
PASS if missing inputs, contradictions, or blockers are surfaced explicitly.
AMBER if some gaps are hinted at but not actionable.
FAIL if major missing information is ignored.

### 5. Structured next step or recommendation
PASS if the result ends with a reproducible recommendation, next step, or patch list.
AMBER if the result is structured but incomplete.
FAIL if the output stops at narrative commentary.
