# Source Routing for `design-upgrade-loop`

Use this file to choose strong external references without overfitting to one source.

## 1) Awwwards
Best for:
- premium visual language
- hero hierarchy
- typography treatment
- motion direction
- immersive but controlled surface ideas

Use when:
- the user wants a higher-end, more distinctive look
- the current interface feels flat, generic, or visually outdated

Extract:
- spacing rhythm
- type scale contrast
- hero composition
- restrained motion ideas
- surface layering

Avoid:
- over-animating dashboards or enterprise tools
- borrowing decorative patterns that hurt readability

## 2) DesignRush
Best for:
- enterprise information architecture
- section ordering
- explanatory UX
- conversion-friendly layout clarity

Use when:
- the target is corporate, SaaS, B2B, executive, or report-like
- the design problem is not beauty alone, but “what should be seen first?”

Extract:
- section sequencing
- headline hierarchy
- content blocks
- CTA rhythm
- enterprise-safe visual emphasis

Avoid:
- turning an operational dashboard into a marketing site

## 3) Web Design Awards
Best for:
- category-filtered and technology-filtered scanning
- balanced inspiration across creativity, usability, and performance-minded execution
- quick comparison by industry or stack

Use when:
- you need category-specific references (for example logistics, transportation, SaaS, technology)
- you want to sanity-check whether a direction is modern but still practical

Extract:
- layout density
- card systems
- stack-compatible patterns
- responsive interaction patterns

Avoid:
- assuming the highest-scoring visual is the best fit for enterprise readability

## 4) WebAward
Best for:
- conservative enterprise evaluation
- final sanity-check for usability and copy clarity
- judging the design against practical review criteria

Use when:
- the target is stakeholder-facing, executive-facing, or highly conservative
- you need a backstop against style drift

Extract:
- usability checks
- copy clarity
- ease-of-use issues
- enterprise-safe scoring language

Avoid:
- using it as the main source for modern motion or expressive visuals

## 5) Surface-specific routing

### Webpage / landing / product page
Priority:
1. Awwwards
2. DesignRush
3. Web Design Awards
4. WebAward

### Dashboard / control tower / data-heavy UI
Priority:
1. DesignRush
2. Web Design Awards
3. Awwwards (selective)
4. WebAward

### Document-surface / report-like / word-style page
Priority:
1. DesignRush
2. WebAward
3. focused web search for executive report layouts
4. Awwwards only for typography/surface inspiration, not motion

## 6) Minimum benchmark rule
Before proposing any redesign:
- capture at least 3 current references
- use at least 2 different sources
- explain why each is relevant
- list one pattern to transfer and one pattern to avoid

## 7) Anti-copy rule
The output must be a synthesis.
Never clone one site’s layout, palette, illustration style, or microinteraction language directly.
